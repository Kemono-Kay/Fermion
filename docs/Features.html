(Planned) Feature List

  • Ease of use

    • Updates are detected automatically.
    • Chat logs are saved automatically and can be imported from F-Chat.
    • When the app starts, everything is resumed the way it was when the app closed.
    • Status messages and ads can be saved for reuse.
    • Previously used eicons can be autocompleted when typing.
    • Commands can be autocompleted and have syntax hints.
    • The application can be minimized to the system tray.
    • Profiles can be viewed inside the application.
    • Characters that don't exist are marked differently from characters that have a default avatar (If I can get that to work)
    • The strength of your connection is estimated, so you can see if you're about to disconnect.
    • Information is cached, so you don't need a connection to use most features.
    • Ads can be posted to multiple channels at once (so no more copying and pasting between channels).
    • Ads can be queued up and posted multiple times, with a customizable delay.
    • A font is included to ensure a consistent user experience when using emoji.
    • The client keeps track of unread in addition to unanswered replies.
    • Channels can be joined as if they are a single channel so you can view related channels at the same time.

  • Searching

    • When searching through channels and/or users, extensive filtering and sorting capabilities are present.
    • Fuzzy search can be used with results sorted by relevancy.

  • Customizability

    • Channels and PMs can be organised in folders and placed in any order.
    • Channels can be split into separate ads and chat channels for organisation purposes.
    • Users can be grouped into as many different groups as you want, and can be hidden, highlighted, recoloured, given different icons, etc. in any way you wish. Default user groups for gender and status are also available for this purpose.
    • Notifications can be customised any way you wish, including whether to play sound, display desktop notifications and more.
    • Many types of notifications can be outright hidden (status change messages, log in/out messages, etc.)
    • You can optionally hide your typing status from users.
    • Words can be blacklisted so messages and/or ads will not show up and/or generate notifications.
    • Settings can apply globally or per character.
    • Visibility of icons, eicons, images and more can be toggled.
    • Timestamp format of messages can be extensively customised.
    • User avatars can be shown or hidden in the user list.
    • You can locally attach a memo or flair to a user and/or channel.
    • You can automatically respond to incoming messages.

  • Themes

    • Custom CSS themes.
    • Themes may change the layout of the app.
    • Themes may change the icons the app uses.
    • Themes may change the tab order of elements.
    • You may apply custom CSS in addition to a theme.
    • You may apply multiple themes at once, providing support for modular themes.
    • There is a custom "theme editing" mode for easy customisation.

  • Styling messages

    • BBCode parsing for the following tags: [b], [i], [u], [s], [sup], [sub], [color], [url], [user], [icon], [eicon], [noparse], [session], [li], [ol], [ul], [img], [big], [small], [left], [right], [center], [justify], [code], [heading], [collapse], [quote], [indent], [hr].More details...
    • Markdown-inspired syntax as an alternative way of styling messages.
    • A rich text editor to write and preview messages in one go.
